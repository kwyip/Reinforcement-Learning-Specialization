It is important you follow the instructions in the notebooks. In particular, we will tell you which random seeds to use, and where possible, provide very specific advice about how to call the random number generator in your agents. If you follow those instructions carefully, you should get exactly the same solution – the same learning curve – as our solution. But mistakes happen, and things are complicated. It is possible your agent is correct, but because an extra call to the random number generator or a different seed, your results might differ significantly from ours.

But don’t worry, we have designed a very robust grading system! Even if you don’t precisely follow our instructions regarding randomness, it should be ok (though it’s better if you do follow the instructions).  If your agent is correct, then our graders will agree and you will pass the assignment. However, it is actually very tricky to build such a robust graders for thousands of learners. We did it, but it will make the submission process slightly more complicated. But it’s not too much extra work for you, and it’s totally worth it!

The process is described in each notebook. Here we provide an FAQ to help you out if the grading process is a pain in your agent. Check it out below. We will add additional questions to the FAQ as they come up in the forum.

  1. The submission procedure is complicated. Can you go through it again?

  • Step 1: Code up your solutions and run the experiment cells.

  • Step 2: Running the experiment cells generates stores your data-files in a `results/` folder, and zips it into `results.zip`.

  • Step 3: ‘Submit’ the notebook by clicking on the blue button on the top right of the notebook.

  • Step 4: To access this `results.zip` that you need to submit as well, navigate to ‘File -> Open’. This opens the directory/tree view of the notebook.

  • Step 5: Select `results.zip` and click on the ‘Download’ button.

  • Step 6: On the Programming Assignment page, go to the ‘My submission’ tab.

  • Step 7: Here you should see the partial marks you obtained on submitting the notebook. Click on ‘+ Create Submission’.

  • Step 8: Click on ‘C2M4 Data-file Grader’ to upload your latest downloaded `results.zip` file.

  • Step 9: Click on ‘Submit’. Within a few seconds, you should be able to see the total marks you obtained in the notebook and the experiments.

2. Where is my `results.zip` folder?

In the notebook, go to ‘File -> Open’. In the directory/tree view of the notebook, you should be able to see `results.zip` among all the other files and folders like `results`, `images`, `Planning_Assignment-v*.ipynb`, etc.

3. I can’t see my `results.zip` folder even after opening the directory/tree view of the notebook. What do I do?

Did you run the experiment cells? The `shutil.make_archive` command in those creates the zipped file of the `results` folder, and hence the experiment cells need to be executed in order to generate (or update) the zip file.

4. I submitted my `results.zip` folder but am getting the feedback from the grader  as: " Dyna-Q: Submission file not found. Please make sure to run all codeblocks.,  Dyna-Q+: Submission file not found. Please make sure to run all codeblocks.

Check the results.zip folder that you are submitting – it should have the following hierarchy:

- results.zip

---- Dyna-Q_planning_steps.npy

---- Dyna-Q+.npy

Note that there is no ‘results’ folder inside this zip file. This is important if you are zipping the results folder yourself. The `shutil` command in the experiment cells of the notebook generate the hierarchy outlined above, which is what the grader expects. If you are zipping the folder yourself, make sure to use the ‘j’ flag with the zip command: zip -jr results.zip results/

This creates a ‘results.zip’ from the contents of the ‘results/’ folder. Without the ‘j’ flag, it’ll create a ‘results’ folder inside the zip, which is not what we want.
